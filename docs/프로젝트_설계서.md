# EgAg (이그에그) — 프로젝트 설계서

> AI와 사용자가 1획씩 번갈아 그림을 완성하는 협업 드로잉 서비스

## 기술 스택

| 영역 | 기술 |
|------|------|
| 프론트엔드 | React 19 + Vite + TypeScript |
| 캔버스 | Konva.js + react-konva |
| 차트 | Recharts (AreaChart, LineChart) |
| 아이콘 | Lucide React |
| 백엔드 | Spring Boot 3.4.3 + Java 25 |

| AI | LangChain4j 1.0.0-beta2 + GPT-4o Vision |

| 결제 | 포트원 (PortOne) — 카드 / 카카오페이 / 토스페이 |
| 상태관리 | Zustand |
| HTTP 클라이언트 | Axios |

## 핵심 용어 정의

- **1획(stroke)**: 사용자가 마우스/터치를 누르고(pointerdown) 떼는(pointerup) 한 번의 연속 드로잉
- **1턴**: 사용자 1획 + AI 1획으로 구성
- **세션**: 캔버스 진입부터 완성까지의 전체 드로잉 과정
- **토큰**: 그림 1회 시작에 필요한 재화. 신규 가입 시 무료 지급, 이후 결제로 충전

## 캔버스 핵심 로직

### 턴 흐름
1. 사용자 pointerdown → 새 stroke 생성
2. pointermove → points 배열에 좌표 추가
3. pointerup → 1획 완료 → `POST /api/canvas/{id}/stroke`
4. 서버: 캔버스 이미지 → GPT-4o Vision → AI 획 좌표 반환
5. 프론트: AI 획 애니메이션 렌더링
6. 다음 사용자 차례

### 색상 구분
- 사용자 획: 검정 (#000000)
- AI 획: 파랑 (#3B82F6)

## 인증 흐름

### JWT
- Access Token: 15분 만료, `Authorization: Bearer` 헤더로 전송
- Refresh Token: 7일 만료, DB에 저장
- 자동 갱신: Axios 인터셉터 처리

### 카카오 OAuth
1. `GET /api/auth/kakao` → 카카오 로그인 페이지 리다이렉트
2. `/oauth/callback` → Access/Refresh 토큰 발급
3. 최초 로그인 시 닉네임 미설정 → `/kakao-onboarding` 강제 이동

## 코딩 컨벤션

### 백엔드 (Java/Spring Boot)
- 패키지: `com.egag.*`
- Entity: 단수형 (User, Artwork)
- Repository: `~Repository`
- Service: `~Service`
- Controller: `~Controller`
- DTO: `~Request` / `~Response`

### 프론트엔드 (TypeScript/React)
- 컴포넌트: PascalCase (`DrawingCanvas.tsx`)
- 훅: `use~` (`useCanvas.ts`)
- API 함수: camelCase (`getArtwork`)
- 스토어: `use~Store` (`useAuthStore`)
- 타입: PascalCase (`ArtworkResponse`)
